## 概述

FlecBlog 提供了官方一键安装脚本，通过交互式问答完成原生部署（PM2 + PostgreSQL），相比 Docker 部署具有更低的资源占用和更直接的系统集成。

## 系统要求

- **操作系统**：Linux（Ubuntu/Debian/CentOS 等）
- **权限**：root 或 sudo 权限

## 快速开始

```bash
curl -fsSL https://install.talen.cyou | sudo bash
```

<details>
<summary><strong>如何迁移已有部署？</strong></summary>

在运行脚本前，将以下目录和文件准备到安装目录（`/opt/flecblog`）：

| 准备 | 作用 |
|:---: |:---: |
|.env.backup |取得 JWT_SECRET，保持存在用户的登录状态 |
|uploads/ |采用本地上传时，所有文件的存放位置 |
|backup.sql |数据库 |

```bash
# 创建安装目录
mkdir -p /opt/flecblog

# 迁移配置，找到 docker-compose.yml 存放目录，通常 .env 文件也放在这里
cp <存放目录>/.env /opt/flecblog/.env.backup

# 迁移上传文件，根据 docker-compose.yml 内容找到挂载目录，通常是 /srv/flecblog
cp -r <挂载目录>/uploads /opt/flecblog/

# *导出数据库（数据库在容器内，根据 docker-compose.yml 替换容器名，通常是 flec_postgres，另外按需修改数据库名称和用户名）
docker exec <容器名> pg_dump -U postgres -d postgres > /opt/flecblog/backup.sql
# *导出数据库（数据库在宿主机，按需修改数据库名称和用户名）
sudo -u postgres pg_dump -d postgres > /opt/flecblog/backup.sql

# 停止旧服务，避免后续操作占用端口
cd <docker-compose.yml 存放目录> && docker compose down
```

</details>

## 安装流程

脚本运行后会自动完成以下步骤：

1. 检测系统环境和依赖工具
2. 检测并安装 PostgreSQL（如未安装）
3. 交互式配置（数据库连接、JWT 密钥、端口、Nginx/SSL 等）
4. 确认配置信息
5. 下载 Server 二进制文件
6. 构建 Blog 前端应用
7. 导入数据库备份（如有 `backup.sql`）
8. 生成配置文件并启动服务
9. 健康检查并显示访问地址

整个过程只需按提示输入配置信息，其余操作全自动完成。

## 常见问题

### 配置 PostgreSQL 远程访问

如果数据库在其他服务器上，需修改 PostgreSQL 配置：

```bash
# 编辑 postgresql.conf
listen_addresses = '*'

# 编辑 pg_hba.conf，添加
host    all             all             0.0.0.0/0               md5

# 重启服务
sudo systemctl restart postgresql
```

### 端口被占用

使用 `ss` 或 `netstat` 查看占用进程：

```bash
ss -tlnp | grep :8080
```

停止占用端口的进程或更换端口。

### 服务启动失败

查看 PM2 日志：

```bash
pm2 logs
```

常见原因：

- 数据库连接失败（检查 `.env` 中的 `DB_*` 配置）
- 端口被占用
- 文件权限问题（确保 `/opt/flecblog` 目录可写）

### 清理迁移残留

迁移完成并确认新服务正常后，清理旧 Docker 资源：

```bash
# ⚠️ 仅在确认数据已成功迁移后执行
cd <原 docker-compose.yml 存放目录> && docker compose down -v

# 清理迁移文件（可选）
rm -f /opt/flecblog/.env.backup /opt/flecblog/backup.sql

# 清理 flecb 安装器（可选，如之前使用过旧版安装器）
sudo rm -f /usr/local/bin/flecb
rm -rf ~/.flecblog
```

## PM2 命令

安装完成后，可使用以下命令管理服务：

```bash
# 查看所有进程状态
pm2 status

# 查看日志（实时）
pm2 logs

# 查看特定进程日志
pm2 logs flec_server
pm2 logs flec_blog

# 重启所有服务
pm2 restart all

# 重启特定服务
pm2 restart flec_server
pm2 restart flec_blog

# 停止服务
pm2 stop all

# 删除进程
pm2 delete all

# 保存进程列表（开机自启）
pm2 save

# 生成系统启动脚本
pm2 startup
```

## 文件结构

安装完成后，目录结构如下：

```
/opt/flecblog/
├── .env                     # 环境变量配置
├── ecosystem.config.js      # PM2 进程配置
├── flec-server              # Server 二进制
├── blog/                    # Blog 前端源码
│   ├── .nuxt/
│   ├── .output/
│   ├── node_modules/
│   └── 源码
└── uploads/                 # 上传文件
```

## 更新升级

登录管理端 → 系统管理 → 系统信息，点击「检查更新」后一键升级。

## 卸载

```bash
# 停止并删除 PM2 进程
pm2 delete all

# 删除安装目录
sudo rm -rf /opt/flecblog

# 可选：清理数据库（按需修改用户名）
sudo -u postgres psql -c "DROP DATABASE postgres;"
```
